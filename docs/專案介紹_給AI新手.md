# 智慧課程助教系統 — 從零開始的完整介紹

> 這份文件假設你**完全沒碰過 AI**。我們會用生活化的比喻，一步步帶你看懂這個專案在做什麼、
> 用到哪些技術、以及它怎麼運作。讀完你就能跟別人解釋這整套系統。

---

## 0. 一句話總結

**這是一個「只懂你課程教材」的 AI 助教**：你把課堂的 PDF 丟進去，它就能根據教材回答問題、
整理重點、出考題，還能**根據你答錯的觀念，自動出下一份測驗來幫你補強**。

比喻：它就像一個「**只讀過你這門課講義的學霸同學**」，你問什麼它都翻講義回答你，
而且不會亂掰課本裡沒有的東西。

---

## 1. 它想解決什麼問題？

你可能用過 ChatGPT。它很厲害，但有三個問題：

1. **它沒讀過你的課程教材** → 問課堂細節，它不知道。
2. **它會「一本正經地亂講」** → 這在 AI 圈叫**幻覺（Hallucination）**，答案聽起來很對其實是錯的。
3. **它不知道你老師怎麼教** → 回答可能跟上課內容不一致。

這個系統的目標就是：**讓 AI 的回答「有憑有據」，只根據你上傳的教材講話，還能標出處在第幾頁。**

---

## 2. 先搞懂幾個 AI 名詞（白話版）

別怕，這些名詞背後的概念其實很直覺。

### 🧠 LLM（大型語言模型 Large Language Model）
就是 ChatGPT、Gemini 這類「會講人話、會回答問題」的 AI。
**比喻**：一個讀過半個網路、超會聊天的人。但它的知識是「訓練時」記下來的，**不知道你的課程教材**。
本專案用的是 Google 的 **Gemini**。

### 👻 幻覺（Hallucination）
LLM 不知道答案時，會「自信地編一個」。
**比喻**：考試不會寫，硬掰一個答案還寫得煞有其事。我們要想辦法**防止它亂掰**。

### 📚 RAG（檢索增強生成 Retrieval-Augmented Generation）— 本專案的核心
這是解決上面問題的關鍵招式。拆成兩步：
1. **檢索（Retrieval）**：先從你的教材裡，找出跟問題最相關的幾段文字。
2. **生成（Generation）**：把這幾段文字連同問題一起交給 LLM，叫它「**只根據這些內容回答**」。

**比喻**：考試從「閉卷」變成「**開卷**」。先幫 AI 翻到課本對的那幾頁，再叫它照著答 →
答案就有根據、不會亂掰，還能告訴你出自第幾頁。

### 🔢 Embedding（向量嵌入）
電腦看不懂文字，只看得懂數字。Embedding 就是把一段文字**變成一串數字**（叫「向量」），
而且**意思相近的文字，數字也相近**。
**比喻**：幫每段文字標一個「語意 GPS 座標」。「貓」和「小貓」座標很近，「貓」和「汽車」座標很遠。
這樣電腦就能用「座標距離」判斷哪段教材跟你的問題最相關。

### 🗄️ 向量資料庫（Vector Database）
專門存放上面那些「向量座標」、而且能**飛快找出最相近的幾段**的資料庫。
本專案用的是 **ChromaDB**。
**比喻**：一個按「語意座標」排好的超大書櫃，你一問問題，它瞬間抽出最相關的幾頁。

### ✍️ Prompt / Prompt Engineering（提示詞工程）
**Prompt** = 你給 AI 的指令。**Prompt Engineering** = 把指令寫得好，讓 AI 表現更好。
本專案就用它來下死命令：「**只根據以下教材回答，沒有的話就說不知道，不准亂掰**」，
這就是防幻覺的關鍵。

### 🤖 Agent / Workflow / 任務分流（用 LangGraph）
使用者可能想「問問題」、「要摘要」、或「要考題」——這是三種不同任務。
系統需要先**判斷你想做哪一種**，再走對應的流程。這個「指揮中心」用 **LangGraph** 做。
**比喻**：醫院的「**分診台**」，先看你哪裡不舒服，再把你導去對的科別。

## 3. 這個系統實際怎麼運作？

分成兩個階段：

### 階段一：把教材「吃進去」（上傳時做一次）
```
你上傳 PDF
   │
   ▼
① 切割(Chunking)：把長文件切成一段一段（太長 AI 讀不動）
   │
   ▼
② 向量化(Embedding)：每段文字算出「語意座標」
   │
   ▼
③ 存進向量資料庫(ChromaDB)，並記住每段來自哪個檔、第幾頁
```

### 階段二：回答你的問題（每次提問時）
```
你問：「什麼是 RAG？」
   │
   ▼
① 分診台(LangGraph) 判斷這是「問答」任務
   │
   ▼
② 檢索：把你的問題也算成座標，到向量庫找最相近的幾段教材
   │
   ▼
③ 組裝：把「找到的教材 + 你的問題 + 嚴格指令」一起給 Gemini
   │
   ▼
④ Gemini 根據教材生成答案，並附上「引用來源：xxx 第N頁」
```

## 4. 三大功能

| 功能 | 你怎麼用 | 背後做什麼 |
|---|---|---|
| **課程問答** | 「什麼是 RAG？」 | RAG 檢索 → 回答 + 標來源 |
| **重點摘要** | 「幫我整理第 3 章重點」 | 檢索該章 → 叫 AI 條列摘要 |
| **適性測驗** | 「幫我出 5 題選擇題」 | 檢索教材 → 叫 AI 出題 + 附答案解析 |

---

## 5. ⭐ 最大特色：「讓你變強」的適性測驗 + 成效量化

一般出題工具出完就沒了。這個系統多做了一件事：**幫你針對弱點補強，還用數據證明你有進步。**

運作邏輯：
```
你第一次作答測驗
   │
   ▼
系統記下你「答錯的觀念」（例如：記憶體階層、CPU 結構）
   │
   ▼
下一份測驗 → 自動「針對這些弱點觀念」出題
   │
   ▼
反覆練習 → 弱點慢慢被補強
```

然後在「📊 學習成效分析」頁，用四個指標**量化你到底有沒有變強**：

| 指標 | 意思 |
|---|---|
| **正確率趨勢** | 每次測驗分數的變化曲線 |
| **進步幅度** | 最新一次正確率 − 第一次正確率 |
| **弱點克服率** | 一開始答錯的觀念，後來答對的比例 |
| **概念掌握度** | 每個觀念你的累積正確率 |

還能把這些**匯出成 CSV / 報告檔**，放進書面報告。

## 6. 用到的技術積木一覽

把這套系統想成用樂高拼起來的，每塊負責一件事：

| 積木 | 角色 | 比喻 |
|---|---|---|
| **Streamlit** | 做出網頁介面 | 系統的「門面、櫃台」 |
| **Gemini** | 大型語言模型 | 會回答的「大腦」 |
| **ChromaDB** | 向量資料庫 | 按語意排好的「書櫃」 |
| **HuggingFace Embedding** | 把文字轉向量 | 標「語意座標」的工具 |
| **LangChain** | 串接整個 RAG 流程 | 把各積木接起來的「水管」 |
| **LangGraph** | 任務分流指揮 | 「分診台」 |
| **SQLite** | 存對話與作答紀錄 | 系統的「記事本」 |

## 7. 程式檔案地圖（每個檔在幹嘛）

```
course-ta/
├─ app.py              ← 網頁介面（你看到、點擊的所有東西）
├─ Dockerfile         ← 部署用：教雲端怎麼把這專案打包跑起來
├─ requirements.txt   ← 需要安裝的套件清單
├─ README.md          ← 專案說明（也含 HF 部署設定）
└─ src/               ← 核心程式
   ├─ config.py        ← 各種設定（用哪個模型、資料夾在哪）
   ├─ ingest.py        ← 把 PDF 切割 + 向量化存庫（階段一）
   ├─ vectorstore.py   ← 管理向量資料庫 ChromaDB
   ├─ llm.py           ← 連接 Gemini
   ├─ workflows.py     ← 問答／摘要／出題的「指令」怎麼下
   ├─ graph.py         ← LangGraph 分診台：判斷任務 + 導流
   ├─ quiz.py          ← 適性測驗：出結構化題目、計分、依弱點出題
   ├─ analytics.py     ← 學習成效計算 + 匯出報告
   └─ db.py            ← SQLite：存對話與作答紀錄
```

## 8. 它跑在哪？（架構）

**重點：程式碼只有一份，只是放在不同地方 / 在不同地方執行。**

```
   你的電腦（開發）  ──push──▶  GitHub（原始碼倉庫，存檔）
        │                          │
        │ 同一份程式在兩處執行        │ push 後自動部署 ↓
        ▼                          ▼
   本機 localhost:8501        Hugging Face Space（線上 app）
   （只有你看得到，開發測試用）   （公開網址，給別人測，不用你開電腦）
```

- **GitHub**：https://github.com/edge121212/course-ta — 存程式碼。
- **Hugging Face Space**：https://huggingface.co/spaces/edge121212/course-ta — 真正在線上跑、給人用的版本。
- **自動同步**：設了一個機制，只要 `git push` 到 GitHub，**HF 就會自動更新**，不用手動部署。
- 這兩邊的「資料」（教材、作答紀錄）是**各自獨立**的，不互通。

## 9. 怎麼用？

### 一般使用者（線上版）
1. 打開 HF 網址。
2. 左側貼上**自己的 Gemini API key**（每個人用自己的，免費額度，取得：https://aistudio.google.com/app/apikey）。
3. 上傳課程 PDF → 按「匯入教材」。
4. 開始問問題 / 出測驗 / 看成效分析。

> 為什麼要自己貼 key？因為呼叫 Gemini「大腦」是要透過金鑰的，每個人用自己的，互不影響。

### 自己在電腦跑（開發者）
```bash
python3 -m venv .venv          # 建一個獨立的 Python 環境
source .venv/bin/activate
pip install -r requirements.txt # 安裝所有套件
streamlit run app.py           # 啟動，瀏覽器開 localhost:8501
```
